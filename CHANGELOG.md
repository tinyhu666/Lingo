# 更新日志

本文件记录客户端面向用户的重要变更，按版本倒序维护。

写法约定：

1. 面向用户表达，优先说明体验变化与业务价值，避免偏实现细节的工程表述。
2. 每个版本默认按“新增 / 优化 / 修复”三个角度组织；没有对应内容时可保留“暂无”。
3. 同一条文案尽量只表达一个重点，便于同步到 GitHub Release、官网与客户端内展示。

## [0.3.18] - 2026-03-24

### 新增

- 升级品牌资源生成链路，接入新的高清品牌图标，后续客户端、安装包与官网素材可以保持同步更新。

### 优化

- 调整桌面端自动更新源策略，优先使用 GitHub Release 自带的 `latest.json` 与安装包地址，在镜像同步期间也能更快感知新版本。
- 优化 COS 镜像发布顺序，优先同步 updater 关键文件与 `latest.json`，缩短发版后更新信息刷新到客户端的等待时间。
- 统一客户端、安装包与 `lingo.ink` 的品牌图标与分享素材，提升跨端视觉一致性。

### 修复

- 修复 COS 镜像 `latest.json` 尚未同步时，客户端仍可能提示“已是最新版本”的问题，更新结果现在会更准确地反映真实发布状态。

## [0.3.17] - 2026-03-24

### 更新日志

1. 修复桌面客户端四角仍会露出多余斜角边影的问题，将桌面外层光效和应用壳体渐变收回到圆角裁切区域内，窗口边缘显示更干净。
2. 修复 Windows 打包链路继续复用旧图标缓存的问题，品牌资源生成和 Tauri 构建前都会主动清理历史 `icon.ico` / `icon.icns` / `icon.png` 缓存，确保新桌面图标随版本正确更新。

## [0.3.16] - 2026-03-24

### 更新日志

1. 统一客户端、安装包、官网与社交分享图的品牌图标，`icon.svg` 现已成为唯一图形源，视觉识别更一致。
2. 重建 Windows、macOS、iOS、Android 等全平台打包图标，并同步更新应用内标题栏图标、浏览器 favicon 与官网品牌资产。
3. 新增可重复执行的品牌资产生成脚本，后续更换图标时可一次性同步客户端与官网出货资源，减少手工漏改风险。

## [0.3.15] - 2026-03-20

### 更新日志

1. 修复桌面客户端四个角的透明残留与圆角外多余背景，统一窗口、根容器与应用壳体的圆角裁切边界。
2. 重整主页五块核心模块的栅格比例、块高和留白，顶部三张配置卡与底部说明区的节奏更稳定。
3. Windows 桌面窗口改为实体背景色，不再依赖透明窗体承载外层圆角，边缘显示更干净。
4. 提升设置写入、版本检测与前端日志上报的稳健性，补齐空返回、序列化失败和长消息截断场景。
5. 更新客户端发布元数据读取链路，继续优先使用 COS 镜像的 `latest.json`，减小更新信息获取失败的概率。

## [0.3.14] - 2026-03-18

### 更新日志

1. 恢复桌面客户端更紧凑的布局密度，主页、翻译风格、常用语、关于等页面在固定窗口下更接近 0.3.8 的视觉节奏。
2. 修复剪贴板快捷键翻译失效问题：桌面端重新支持内置默认翻译代理地址回退，未显式配置环境变量时也能请求服务端翻译。
3. 重建免费服务端翻译代理链路，默认接入 SiliconFlow 的 `deepseek-ai/DeepSeek-V3`，并补充超时、空响应、非 JSON 返回时的稳定错误信息。
4. 新增翻译代理部署工作流与 Supabase 函数配置，发布构建可注入默认后端地址与匿名公钥，避免正式包因缺失后端配置而不可用。
5. 统一中英俄 README 与应用内核心文案，明确 Lingo 是“游戏内的嘴替”，首屏价值表达更直接。
6. 修复左侧导航底部版本信息与导航项左边距不一致的问题，侧栏整体对齐更稳定。
7. 修复前端版本常量与安装包版本脱节的问题，应用内显示版本现已跟随包版本统一生成。

## [0.3.13] - 2026-03-18

### 更新日志

1. 调整 Windows 发版流程，在未配置代码签名证书时也允许继续生成并发布安装包，避免个人开发者被签名成本卡住发版节奏。

3. 保留已配置签名时的 Authenticode 校验逻辑；若当前为无签名发布，工作流会明确输出警告而不是静默失败。
