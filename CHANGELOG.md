# 更新日志

本文件记录客户端面向用户的重要变更，按版本倒序维护。

## [Unreleased]

### 变更规范
- 仅记录用户可感知的改动，避免重复描述实现细节。
- 使用分组结构：`新增`、`优化`、`修复`。
- 不写“版本升级到 x.y.z”一类占位条目。

## [0.3.2] - 2026-03-11

### 优化
- 规范发布日志来源：改为从 `CHANGELOG.md` 按版本提取，发布文案结构统一、可维护。
- 更新日志文案收敛为简洁分组格式，避免冗余描述和重复信息。

### 修复
- 修复 GitHub Release 自动日志与手写内容混合导致的重复问题。
- 修复发布文案中“版本升级到 x.y.z”这类无效条目残留问题。
- 修复首页与关于页信息类图标风格不一致问题，统一为 `CircleInfo`。

## [0.3.1] - 2026-03-11

### 优化
- 统一客户端圆角体系，修复外边框直角暴露与模块圆角不一致问题。
- 顶栏去除多层边框/背景叠加，窗口操作区视觉更简洁。
- 侧栏移除重复品牌区与“导航”字样，整体对齐与层级更清晰。
- 统一“翻译风格”命名，移除“模式”后缀，文案表达一致。

### 修复
- 修复“关于”导航图标语义不匹配问题，替换为信息类图标。
- 修复应用内与系统图标显示比例异常，提升 Dock/任务栏清晰度与完整性。
- 修复更新说明显示策略：仅在检测到新版本时展示，避免重复历史日志干扰。
