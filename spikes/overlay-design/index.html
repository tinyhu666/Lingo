Lingo · Incoming Overlay · Design Variants

Same 6 incoming messages rendered three ways. All variants assume a 380px-wide overlay panel anchored top-left or bottom-left, semi-transparent, pinned over the game. Background gradient simulates "game in progress" — not a Lingo asset.

Densest, closest to in-game chat. One rounded panel, tight rows. Sender colored by scope. Original prefixed with "orig · " in faded grey.
Pros: scans like native chat, least screen space, lightest GPU.
Cons: wall-of-text in team fights, color is the only scope cue (a11y risk).

Open this file directly in a browser: open spikes/overlay-design/index.html. Mock data uses 6 incoming messages with EN, RU originals translated to ZH. All variants share identical data so layout differences are directly comparable.

Final overlay window will be: always-on-top, click-through (not focusable), fade-out after 8s per message, max 6 visible rows, draggable header. Width 380px is the recommended default.
